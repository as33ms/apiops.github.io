MANIFESTO

VALUES

APIOps values are effectively captured in the Agile Manifesto. APIOps extends the "People over Process over Tools" and urges a mutual collaboration between the API developer, System Operator and 3rd Party Service Developer.

PRINCIPLES

APIOps at the conceptual level is based on Agile's and DevOps principles to focus on effective collaborative design, development and maintenance of APIs to enable extremely fast and flexible service design and development

METHODS

You can use Scrum with operations, Kanban with operations etc. Apart from that, provide methods to collaboratively design APIs for service development utilizing the principles of design first approach.

PRACTICES

Follow practices that provide a low barrier access to reliable APIs, Documentation supporting various use-cases, sandboxes to test and build apps on top of, generated libraries and code examples for 3rd party service developers.

TOOLS

Tools you'd use in the commission of these principles. In the APIOps world there's a lot of emerging tools and evolving standards in provisioning (docker, vagrant), design (RAML, API Blueprint, Swagger), API server code generation (Drakov ), generating OS specific libraries (APIMATIC ), managing APIs (apinf.com, 3scale.com,) and many more.

Do you have improvements or comments to manifesto? Raise an issue in Github or join slack platform. Or both!
